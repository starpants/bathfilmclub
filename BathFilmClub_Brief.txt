# Bath Film Club Website
## Project Brief & Design Specification

---

# Project Overview

Bath Film Club is an existing film discussion group organised through Discord.

The purpose of the website is to provide a public-facing presence for the club, communicate the current film cycle, promote participation via Discord, and maintain an archive of previous themes and film selections.

The website is not intended to be a membership platform and will not support user accounts, voting, nominations, commenting, or discussion. All club activity takes place within Discord.

The website should be lightweight, fast, highly visual, and easy for a single administrator to maintain.

---

# Film Club Process

The club operates in monthly themed cycles.

1. A theme is selected.
2. Members nominate up to two films that fit the theme.
3. Five films are randomly selected from the nominations to create a shortlist.
4. Members vote on the shortlisted films.
5. The two films receiving the most votes become the selected films for discussion.
6. Members watch the films independently before attending the monthly meeting.

The website exists to communicate the current stage of this process rather than facilitate it.

---

# Website Objectives

The website should:

- Promote Bath Film Club.
- Provide information for prospective members.
- Link visitors to the Discord community.
- Display the current theme.
- Display upcoming meeting details.
- Present current film selections.
- Explain how the club works.
- Maintain a searchable archive of previous themes and film selections.

---

# Target Audience

## Existing Members

Members will use the website to:

- Check the current theme.
- View nominated, shortlisted, and selected films.
- Confirm meeting details.
- Browse previous themes.

## Prospective Members

Potential members will use the website to:

- Learn about the club.
- Understand the film selection process.
- Explore previous themes and discussions.
- Join the Discord server.

---

# Site Structure

The website should consist of three content levels.

## Homepage

Displays the current cycle.

## Archive

Provides access to all previous themes.

## Theme Detail Page

Provides a complete record of an individual theme.

No dedicated film pages are required for the initial release, although the architecture should allow for them in the future.

---

# Content Management

The website should not contain any administration functionality.

Specifically:

- No user accounts
- No login system
- No passwords
- No member management
- No online nominations
- No online voting

A separate local administration tool will be used by a single administrator.

---

# Local Administration Tool

A local-only administration application should be used to manage site content.

The tool should allow the administrator to:

- Create and edit themes
- Manage meeting details
- Search TMDb
- Select films from TMDb
- Assign film status:
  - Nominated
  - Shortlisted
  - Selected
- Manage archive entries
- Generate content files for the website

The public website should consume generated data files during its build process.

---

# Film Data

Film metadata should be retrieved automatically from TMDb.

The administrator should only need to identify a film once.

The website should automatically populate:

- Poster image
- Film title
- Release year
- Runtime
- Genres
- Synopsis
- Director
- Producers
- Cast
- Rating (optional)

---

# Information Architecture

The archive should be organised around themes rather than individual films.

Theme → Film Selection Process → Discussion

This reflects the way the club operates and provides a clear structure for historical content.

- Theme description

## Current Film Selection

This should be the primary visual feature of the homepage.

The current cycle should be displayed as a visual pyramid representing the selection process.

### Selected Films

Display the two films chosen for discussion.

Example:

[Poster] [Poster]

### Shortlisted Films

Display all five shortlisted films, including the two selected films.

Example:

[Poster] [Poster] [Poster] [Poster] [Poster]

### Nominated Films

Display all nominated films, including those shortlisted and selected.

Example:

[Poster] [Poster] [Poster] [Poster] [Poster] [Poster] ...

### Purpose

The pyramid should visually communicate:

Nominations → Shortlist → Selection

without requiring explanation.

The panel should display:

- Poster
- Title
- Release year
- Runtime
- Director
- Producers
- Main cast
- Genres
- Synopsis

Optional:

- Trailer link

- Backdrop image

## How the Club Works

# Archive Specification

The archive should be designed to scale to many years of monthly themes.

The archive should remain compact and easy to browse.

---

## Archive List

Each theme should appear as a collapsible row.

Collapsed state:

Month & Year | Theme Title

Examples:

May 2026 | Time Travel

April 2026 | Neo-Noir

March 2026 | Road Movies

---

## Expanded State

Expanding a row should reveal:

- Theme description (optional)
- Two selected films

Example:

May 2026 | Time Travel

Selected Films

[Poster] [Poster]

[View Theme]

---

## Search

The archive should support searching by:

- Film title
- Theme title
- Director

Search results should display:

- Film title
- Theme
- Year
- Status

Example:

Stalker

Theme: Science Fiction Classics

Status: Selected

March 2024

---

# Theme Detail Page

The theme page serves as the complete historical record of a film cycle.

---

## Theme Information

Display:

- Theme title
- Theme description
- Meeting date
- Meeting venue (optional)

---

## Film Pyramid

The page should reuse the same visual structure as the homepage.

### Selected Films

Two films selected for discussion.

### Shortlisted Films

All five shortlisted films.

### Nominated Films

All nominated films.

This consistency helps users understand both current and historical themes.

---

## Theme Navigation

Theme pages should include a sidebar allowing visitors to browse other themes.

Example:

2026
- Time Travel
- Neo-Noir
- Road Movies

2025
- Folk Horror
- Heist Films
- New Hollywood

The sidebar should remain available while viewing a theme.

Assumptions

* Themes occur monthly.
* Themes are displayed newest first.
* Nominated films are unique.
* The website is read-only.
* Film metadata is sourced from TMDb.
* The archive should remain usable with 20+ years of monthly themes.
* Mobile devices are first-class supported.

# Visual Design Specification

## Design Direction

The visual identity should be derived directly from the Bath Film Club logo.

The site should feel:

- Modern
- Minimal
- Confident
- Editorial
- Cultural rather than commercial

The design should resemble a contemporary arts organisation, film festival, or gallery rather than a traditional cinema website.

Film posters should provide most of the site's visual richness and colour.

---

## Layout

The layout should be strongly grid-based.

Characteristics:

- Clear alignment
- Consistent spacing
- Strong hierarchy
- Minimal decorative elements
- Responsive across desktop and mobile

---

## Typography

### Headings

Font: Space Grotesk

Characteristics:

- Bold
- Geometric
- Contemporary
- Strong visual presence

### Body Text

Font: Inter

Characteristics:

- Highly readable
- Clean
- Neutral

---

## Colour Palette

### Primary Colours

Black

#000000

White

#FFFFFF

### Accent Colour

Red

Used sparingly for:

- Links
- Buttons
- Hover states
- Interactive elements

Suggested value:

#B11226

### Supporting Colours

Neutral greys may be used for:

- Dividers
- Background variation
- Secondary text

---

## Visual Language

The square motif from the Bath Film Club logo should be reused throughout the interface.

Potential applications:

- Section headings
- Navigation indicators
- Archive markers
- Active states
- Decorative structural elements

Example:

■ Current Theme

■ Archive

■ Selected Films

This motif should become a recurring visual signature across the site.

---

## Imagery

Film posters should be the primary imagery throughout the site.

Avoid:

- Stock cinema photography
- Film reels
- Popcorn imagery
- Generic movie-related graphics

The visual focus should remain on:

- Posters
- Typography
- Layout
- Theme content

---

# Technical Goals

The finished website should:

- Be lightweight and fast
- Be responsive on desktop and mobile
- Be inexpensive to host
- Be easy for one administrator to maintain
- Require no public authentication
- Scale comfortably to decades of monthly themes
- Support a large archive of films and themes

The website should feel like a curated film archive and cultural resource rather than a social platform.
